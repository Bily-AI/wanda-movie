# 会员模块旧系统一致性复刻计划

## 阶段 1：服务层补齐

目标：补全旧系统会员模块依赖的真实接口和数据结构。

任务：

1. 扩展 `featureApi.ts` 中的会员类型定义
2. 补齐 Rtime 等级数据的旧字段映射
3. 补齐 W+ 会员状态检测接口
4. 补齐 W+ 权益活动分组查询接口
5. 补齐 W+ 单项领取接口
6. 补齐 W+ 激活兑换接口
7. 统一会员相关错误消息与日志可读性

验收：

- 服务层能独立返回旧页面需要的 Rtime / W+ 数据结构
- 不再把 W+ 强行压成扁平表格数据

## 阶段 2：会员页面结构复刻

目标：把 `MemberView.vue` 从简化版改回旧系统结构。

任务：

1. 恢复 `Rtime / W+` 子页签结构
2. 恢复 Rtime 会员信息卡、签到区、进度区、等级卡片区
3. 恢复 W+ 激活兑换卡、W+ 信息卡、权益分组区
4. 恢复旧系统按钮分布和状态展示
5. 调整样式，使信息层次和旧系统接近

验收：

- 会员页视觉结构和旧系统主布局一致
- W+ 不再使用简化表格

## 阶段 3：交互与真实流程恢复

目标：恢复旧系统真实操作链路。

任务：

1. 恢复 Rtime 单项领取
2. 恢复 Rtime 一键领取
3. 恢复 W+ 单项领取
4. 恢复 W+ 一键领取
5. 恢复 W+ 所有账号批量领取
6. 恢复 W+ 激活兑换
7. 恢复去使用提示逻辑

验收：

- 所有按钮都能走真实接口
- 成功 / 失败提示符合旧系统预期

## 阶段 4：回归与收口

目标：确认会员模块达到旧系统一致性。

任务：

1. 测试无账号、有账号、切换账号
2. 测试签到刷新
3. 测试 Rtime 领取流程
4. 测试 W+ 领取流程
5. 测试 W+ 激活流程
6. 测试批量领取统计结果
7. 检查日志记录和异常提示

验收：

- 会员整页可稳定使用
- 没有明显错位、假数据或缺流程
