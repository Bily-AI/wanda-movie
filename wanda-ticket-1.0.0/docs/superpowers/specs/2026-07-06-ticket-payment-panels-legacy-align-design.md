# 购票页右侧支付三块旧版对齐设计

## 背景

当前购票页右侧 `支付活动 / 支付卡 / 兑换券` 三块已经接上真实数据和勾选链路，但界面结构与旧版打包产物不一致：

- 当前版把三块都做成了可折叠面板，标题区点击会收起或展开。
- 当前版的支付卡、兑换券使用偏新的表格化块样式，视觉密度、标题信息和旧版差异较大。
- 用户这次只要求图二红框里的三块样式与点击勾选逻辑对齐旧版，不要求同步调整 `已选座位`、`全局订单信息` 或购票主区域。

本轮目标是在不扩散改动面的前提下，把这三块恢复到更接近旧版的固定展开形态和交互节奏。

## 目标

- 只调整购票页右侧三块：`支付活动`、`支付卡`、`兑换券`。
- 去掉这三块当前的折叠交互，改为默认展开、稳定展示。
- 保留已接好的真实数据来源和现有 `ticketStore` 选择链路，不重设计状态模型。
- 让点击选择、取消选择、数量显示和空状态更接近旧版界面行为。

## 范围

涉及文件：

- `src/renderer/components/PaymentPanel.vue`
- `src/renderer/components/PayCardList.vue`
- `src/renderer/components/CouponList.vue`
- `src/renderer/views/TicketView.vue`

本轮不改：

- `SelectedSeatList.vue`
- `全局订单信息` 面板
- 选座图、底部操作区、支付提交链路
- `ticketStore` 的金额计算、支付参数拼装和自动匹配核心逻辑

## 方案

### 1. 支付活动面板

- 移除折叠状态和标题区整块点击收起逻辑。
- 保留右上角 `优惠` 开关，继续使用当前 `selectedActivity` 与 `update:selectedActivity` 链路。
- 保留中间活动下拉和右侧活动价展示，但把头部、间距和边框层级收敛到旧版样式。
- 关闭优惠时直接清空当前活动；重新打开时恢复最近一次活动选择，沿用当前组件里已有的 `lastSelectedActivity` 语义。

### 2. 支付卡面板

- 移除折叠状态，面板始终展开。
- 标题区保留 “已选 x / y 张” 统计，样式对齐旧版的紧凑头部。
- 列表继续使用当前 `selectedValues`，点击整行切换选中或取消选中，不新增 checkbox 状态层。
- 收敛表格样式，让行高、留白、选中底色、余额强调更接近旧版，而不是保留当前折叠卡片感。

### 3. 兑换券面板

- 与支付卡保持同样结构：默认展开、标题区显示已选和可兑数量。
- 点击整行切换券选择，沿用当前 `selectedValues` 更新方式。
- 保留 `类型 / 有效期` 信息列，但按旧版压缩视觉密度和层级。
- 空状态文案继续根据 `loading` 区分“加载中”和“暂无可用兑换券”。

### 4. TicketView 接线边界

- 只保留这三块的组件接线变更，不调整其外层布局分区。
- 如果旧版样式需要微调传参，只在 `TicketView.vue` 做最小改动，不引入新的公共配置对象。

## 不做事项

- 不把 `已选座位` 一并改成旧版。
- 不新增“整块右侧统一折叠/展开”之类的新交互。
- 不改支付活动、支付卡、兑换券的真实接口或 `ticketStore` 数据结构。
- 不为了这三块抽新的公共面板组件。

## 验证

先补一条针对三块结构的契约检查，再做实现并跑绿：

- `PaymentPanel.vue` 不再包含折叠用的 `isCollapsed`、`Minus`、`Plus` 交互结构。
- `PayCardList.vue` 不再包含折叠状态，仍保留整行点击选择逻辑。
- `CouponList.vue` 不再包含折叠状态，仍保留整行点击选择逻辑。
- 现有 `tools/check-ui-workbench-layout-contract.mjs` 继续通过，确保没有破坏购票页整体布局契约。
- 最后跑一次项目构建，确认模板和 TypeScript 没有回归。
