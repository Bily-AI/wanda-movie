# 会员模块旧系统一致性复刻设计

## 目标

以旧 Electron 打包产物中的 `VipView` 为唯一标准，对当前项目的会员模块做一比一功能复刻，补齐当前缺失的会员能力、页面结构和交互流程。

本次不做“更现代”的重设计，不做最小补丁，不改变 Wanda 真实接口来源，只按旧系统业务行为恢复。

## 旧系统基准

- 旧页面资源：`D:\bily\win-ia32-unpacked\win-ia32-unpacked\resources\app\out\renderer\assets\VipView-BkMCWbzQ.js`
- 旧样式资源：`D:\bily\win-ia32-unpacked\win-ia32-unpacked\resources\app\out\renderer\assets\VipView-DFErTa5z.css`
- 旧总入口索引：`D:\bily\win-ia32-unpacked\win-ia32-unpacked\resources\app\out\renderer\assets\index.deobf.js`
- 旧模块索引说明：`D:\bily\docs\old-package-index.md`

## 当前问题

当前 `src/renderer/views/MemberView.vue` 只实现了一个简化会员页，和旧系统相比存在明显缺口：

1. `Rtime / W+` 虽有页签，但结构和交互都不完整
2. `Rtime` 只做了单个领取和“取第一个可领项”的简化一键领取
3. `W+` 只展示了扁平权益列表，没有旧系统的分组权益结构
4. 缺少 `W+` 一键领取
5. 缺少“领取所有账号”的批量 W+ 领取
6. 缺少 `W+` 卡号卡密激活兑换
7. 缺少旧系统的等级进度、到期成长值、降级提示等完整信息
8. 缺少旧系统的“去使用 / 跳转”逻辑
9. 页面样式、信息层次和旧系统不一致

## 旧系统真实能力清单

### 1. Rtime 区域

- 加载会员等级权益列表
- 识别当前等级、已超越等级、未达到等级
- 展示成长值进度、下一等级、满级状态
- 展示本月即将到期成长值及降级后的等级提示
- 展示每日签到卡片和签到日历
- 支持单个权益领取
- 支持一键领取当前账号所有可领取权益
- 支持已领取权益“去使用”
- 展示高等级未解锁权益

### 2. W+ 区域

- 检测当前账号是否为 W+ 会员
- 加载 W+ 会员权益活动分组
- 按权益组展示 `rightList`
- 支持单个 W+ 权益领取
- 支持当前账号 W+ 一键领取
- 支持所有已登录账号批量领取 W+ 权益
- 支持 W+ 卡号 / 密码激活兑换
- 支持不同状态的 W+ 权益展示：待领取、已领取、已过期、去使用、需实名认证、生日专享等

### 3. 公共行为

- 账号未登录时展示明确提示
- `userIdentifier` 缺省时使用旧系统默认值 `YYDDJDKYHA`
- 账号切换时自动刷新对应会员数据
- 所有操作都记录日志
- 操作失败时保留已成功加载区域，不做整页清空式失败

### W+

当前 `wplusRows` 的扁平数组模型不够，需要改成旧系统分组模型，至少要能承接：

- 标签
- 副标题

## 交互对齐要求

### Rtime 一键领取

- 旧系统逻辑不是“取第一个可领项”
- 必须遍历当前账号所有可领取权益逐个调用领取接口
- 成功和失败都要统计并反馈结果

### Rtime 单项领取

- 仅对可领取项显示领取入口
- 已领取项优先展示“去使用”样式或使用图标

### W+ 一键领取

- 遍历当前账号所有 `receiveStatus === 1` 的权益
- 成功后刷新当前 W+ 数据

### W+ 所有账号领取

- 遍历所有已登录账号
- 每个账号先查询其 W+ 权益活动
- 对每个账号的可领取 W+ 权益逐项领取
- 最后汇总“成功账号数 / 失败账号数 / 总领取项数”

### W+ 激活兑换

- 必须校验卡号和密码都非空
- 激活结果展示旧系统的成功 / 失败卡片反馈
- 激活成功后自动刷新当前账号 W+ 状态和权益

### 去使用

- 先按旧系统复刻原有提示逻辑：
  - 跳转购票
  - 跳转卖品
  - 跳转外链
  - 普通去使用提示
- 本阶段先保持旧系统提示行为一致，不额外设计新跳转体系

## 实现原则
