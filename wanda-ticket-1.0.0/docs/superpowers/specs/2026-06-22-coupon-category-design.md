# 兑换券分类与统计视图设计

## 目标

兑换券页的“分类管理”和“统计”入口按旧 Electron 包恢复为可用功能，不再只弹提示。

## 旧包依据

- 分类数据使用本地 `categories.json`，不是万达远端接口。
- 分类结构：`{ id, name, couponNames }`。
- 分类筛选按分类关联的 `couponNames` 匹配当前真实兑换券列表里的券名称。
- 统计视图按当前真实兑换券列表里的券名称分组，显示每种券的张数。

## 页面行为

- 启动兑换券页时读取本地 `categories`。
- “分类管理”打开弹窗，可新增、删除分类，并为分类选择多个真实券名称。
- “按分类筛选”使用本地分类筛选当前真实兑换券，不补 mock 券数据。
- “统计”切换为统计表，再点一次回到“明细”。
- 分类配置写入本地数据，关闭重启后保留。
